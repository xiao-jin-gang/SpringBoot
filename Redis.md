## Nosql概述

为什么要用Nosql

 关系型数据库：行、列、表

### Nosql特点

1.方便扩展（数据之间没有关系，很好扩展）

2.大数据高性能（Redis一秒写8万次，）

3.数据类型是多样性的！（不需要事先设计数据库！随取随用！）

4.传统的RDBMS和NoSQL

```
传统的RDBMS
- 结构化组织
- SQL
-数据和关系都存在单独的表中
```

```
NoSql
- 不仅仅是数据
- 没有固定的查询语言
- 键值对存储，列存储，文档存储，图形数据库
- 最终一致性
- CAP定理 和 BASE (异地多活)
- 高性能，高可用，高可扩展性
- .......
```
